# ROG 液金检测工具

## 用途

本工具筛查 CPU/GPU 散热接触异常：支持 CPU 单烤、GPU 单烤和 CPU+GPU 双烤，记录同类型核心温差、温度、功耗、负载、频率和风扇信息，输出原始 CSV、JSON 和 HTML 报告。

它不能在不拆机的情况下确认液金是否物理偏移、覆盖不足或泄漏；结论应理解为“未见明显异常 / 建议复测 / 疑似散热接触异常 / 无法判断”。

## 使用方法

1. 接通原装适配器，将笔记本放在水平硬质桌面。
2. 关闭游戏、渲染和其他硬件监控工具；确认 Armoury Crate/G-Helper 性能档位后，在程序中选择同一档位。
3. 双击 `ROG-LiquidMetal-Inspector.exe` 并允许 Windows UAC。主程序默认请求管理员权限，以便 LibreHardwareMonitor 完整访问传感器。
4. 选择 CPU 单烤、GPU 单烤或 CPU+GPU 双烤，输入室温后开始检测。完整流程约 13 分钟；快速筛查不形成最终结论。
5. 按提示确认适配器、摆放、室温和性能档位条件。程序会把确认状态和规则/档案哈希写入报告。
6. 完成后点击“打开报告”。每次报告在当前版本目录的 `Reports\时间戳\` 下单独保存。

## 判定规则

- P-Core 与 E-Core 分开计算；不会将两种架构核心的正常温差互相比较。
- 同类核心温差 P95 < 15°C：未见明显异常。
- 15–20°C：建议检查环境、后台、电源和性能模式后复测。
- >=20°C 且连续 >=60 秒：疑似散热接触异常；应以同条件完成第二次完整检测。
- 没有每核心温度或 CPU Package Power：无法判断，不应拿温度单独归因于液金。
- 首次完整测试只形成候选结果；同机型、同模块、同档位、相近室温的最近三次中至少两次一致，才输出正式正常或疑似异常结论。
- 温度仍在明显变化、采样有缺口、零负载/掉载、设备不一致或负载/功耗波动过大时，程序会要求复测。

20°C 为社区经验复测阈值，不是 ASUS 官方确诊标准。

## 文件说明

- `ROG-LiquidMetal-Inspector.exe`：主程序，文件版本与界面版本一致。
- `lib\`：LibreHardwareMonitor 运行依赖，仅用于只读传感器访问。
- `规则默认配置.json`：默认流程和阈值说明。
- `Reports\时间戳\samples.csv`：带真实时间戳的原始采样，包含 P/E 核分组和完整 GPU 字段。
- `Reports\时间戳\result.json`：结构化判定结果。
- `Reports\时间戳\summary.html`：可直接打开或交给售后查看的摘要。

## 安全

程序不自动调节风扇、功耗、超频或降压。任何 CPU 负载阶段达到 100°C、或 GPU 达到 90°C并连续 3 秒时，都会自动停止并保留数据。用户停止、传感器失败、压力失败和安全停止均不能输出绿色“正常”结论。
