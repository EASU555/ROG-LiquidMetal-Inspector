# v1.1.0 散热接触异常综合判定规则

## 1. 结论的正确含义

程序检测的是“CPU/GPU 散热接触或整条热路径是否出现异常迹象”，不是直接观察液态金属。软件遥测无法唯一地区分液金迁移、泵出、初始少涂、润湿不良、冷板翘曲、螺钉压力不均、导热垫反力、灰尘堵塞、风扇异常或均热板能力下降。因此报告只允许使用“本次疑似散热接触异常”“建议重复验证”“无法判断”等表述，不应写成“液金偏移已确认”。

ASUS 官方确认 G815 系列 CPU 使用 Conductonaut Extreme，并公布 G815LR 的 RTX 5070 Ti 最高可到 140W；但公开页面没有明确证明该 GPU 的具体界面材料，因此 GPU 模块统一使用“GPU 热路径/接触异常”措辞。

## 2. 为什么不再使用单一温度阈值

Core Ultra 9 275HX 的 Intel 官方最大工作温度为 105°C，PBP 为 55W、MTP 为 160W。PBP、MTP、PL1、PL2、Tau 和 OEM 实际策略不是同一个概念，不能把 55W 或 160W 当作固定稳态目标。现代处理器会主动利用温度余量提升性能，接近温度上限只说明正在受到热约束，不说明液金发生了物理移动。

互联网评测进一步证明绝对值的离散很大：精确 G815LR 样本在 Turbo CPU 长负载中约为 146W/89°C；同 G815 机身的另一个压力负载样本约为 93W/98°C，双烤时 CPU 约为 58W/93°C。这些都可能是正常表现。差异来自压力软件、BIOS、风扇策略、GPU 配置和动态功耗分配。因此 v1.1.0 规定：单一高温只记录为信息项，异常分为 0。

## 3. 五类证据

### 3.1 空间不均（Spatial）

Intel 混合架构不能把 P-Core 与 E-Core 直接比较。程序先在 P-Core 组和 E-Core 组内分别计算温差 P95，再取较大者。Intel 公开资料说明 DTS 在部分平台工作范围内约有 ±5°C 误差，两个传感器最坏方向的相对差可能接近 10°C，因此小温差证据很弱。

- 组内温差 P95 低于 15°C：0 分。
- 15–20°C：15 分，只建议复测。
- ≥20°C 且连续 ≥60 秒：35 分。
- 温差异常时，同一热点核心在 ≥70% 异常样本中重复出现：另加 10 分，但仍属于同一个“空间”类别，不会伪造第二种独立证据。

GPU 只有在驱动真实提供 Hot Spot 时才计算 `Hot Spot - GPU Core`：

- P95 低于 20°C：0 分。
- 20–30°C：15 分。
- ≥30°C 且持续 ≥60 秒：35 分。

这些值是保守的工程复测线，不是 AMD、NVIDIA 或 ASUS 的官方界面材料阈值。RTX 50 系列通常不向通用监控工具提供有效 Hot Spot；字段缺失时程序显示“不可用”，绝不写入 0°C 或把缺失当正常。

### 3.2 高温—低功耗耦合（ThermalResistance）

高温只有和异常低功耗同时出现才支持“在较少发热量下已经撞到热边界”。G815LR 档案采用更保守的 CPU 组合线：CPU 负载成立时，Package 温度 ≥95°C 且 Package Power ≤60W 连续 ≥60 秒，CPU 单烤计 30 分；双烤受功耗分配影响，只计 20 分。

RTX 5070 Ti Laptop GPU 官方子系统功耗实现范围为 60–115W，G815LR 可通过 Dynamic Boost 最高达到 140W。因此 GPU 单烤先要求独显负载 ≥85%，再检查核心 ≥85°C 且功耗 ≤70W 持续 ≥60 秒，计 30 分；双烤只计 20 分。140W 是上限配置，不是每种负载的持续合格线。

如果没有匹配到精确机型档案，程序不使用通用瓦数阈值给异常分，只报告“缺少机型功耗档案”。

### 3.3 随热积累的性能退化（TemporalDegradation）

程序把稳定分析窗口划分为前、中、后三段，比较末段与首段：

- 功耗保持率低于 80%：10 分；低于 65%：20 分。
- 频率保持率低于 85%：7 分；低于 70%：15 分。

保持率只有在温度进入机型高温区间且负载成立时才计分。低负载、测试阶段切换、功耗策略改变造成的下降不会被直接归因于散热。温度斜率绝对值大于 2°C/min 或功耗波动系数过大时，程序降低置信度，因为此时尚未进入可比较稳态。

### 3.4 空闲基线（Baseline）

同机型异常个案出现过空闲 90–98°C、轻负载 95–105°C 和性能明显下降；更换界面材料后温度与性能改善。但这是 n=1 用户自报，不能推导故障率。程序仅在 CPU 平均负载 ≤20% 且空闲温度 ≥80°C 时给 15 分辅助线索。后台任务、风扇静停、静音模式和进风受阻都可能造成相同表现，因此空闲温度永远不能单独给红色结论。

### 3.5 双烤总功耗与配置排除（System / Configuration）

NVIDIA Dynamic Boost 会在 CPU、GPU 与显存之间动态转移功耗，所以双烤不能要求 CPU 和 GPU 同时达到各自单烤上限。程序同时记录 CPU+GPU 稳态芯片功耗和总功耗保持率。负载持续成立、高温背景存在且总功耗保持率低于 75% 时，只给 8 分系统级线索，含义是“整机热饱和或平台限制”，不能定位异常在 CPU 还是 GPU。

GPU 功耗低于机型参考下限但温度不高时，程序记录“更像档位、驱动、适配器或 Dynamic Boost 配置差异”，仅为配置告警，不作为强液金线索。

## 4. 可疑度与独立证据

CPU 和 GPU 分别累加本组件触发分，单组件最高封顶 100。双烤总分取 CPU/GPU 较高者，再加系统级有效线索。为了防止把同一现象重复计数，独立证据数按类别去重；例如“核心温差大”和“固定热点”都属于 Spatial，只算一个类别。

| 输出 | 条件 |
|---|---|
| 本次未见明显散热接触异常 | 可疑度 <25，且数据置信度合格 |
| 异常线索不足以确诊，建议复测 | 可疑度 25–49，或高分但只有一个独立类别 |
| 本次疑似散热接触异常，需重复验证 | 可疑度 ≥50，至少两个独立类别，置信度合格 |
| 本次高度疑似散热接触异常，需重复验证 | 可疑度 ≥70，至少两个独立类别，置信度合格 |
| 无法判断/证据置信度不足 | 传感器覆盖、负载、时长、稳定性或运行完整性不满足要求 |

红色结论不是拆机确诊。建议保持室温、摆放、适配器、BIOS、驱动和性能档位一致，完成三次独立测试并至少复现两次，再决定是否送修。一次绿色结果也不能证明液金围栏、绝缘和实体位置安全；导电液金外泄导致的电气故障可能没有热学前兆。

## 5. 置信度

置信度与可疑度完全分开，避免“数据少所以看起来正常”。满分由以下部分组成：有效样本覆盖 35、分析时长 25、负载稳定性 20、精确机型档案 10、室温有效 5、传感器丰富度 5。

以下情况会降级：

- CPU/GPU 有效样本率低于 90%：该组件不可判断。
- 完整分析稳定窗口不足 240 秒：时长分降低。
- CPU 或 GPU 负载波动 CV >0.15：降低置信度并记录限制项。
- 功耗波动 CV >0.25：置信度减 10。
- 温度趋势绝对值 >2°C/min：置信度减 8。
- 快速筛查置信度最高 55，永远不输出正式正常/异常结论。
- 完整检测的正式结论要求置信度 ≥65。
- 用户停止、安全停止、传感器失败或压力负载失败后，结果不得保持绿色。

## 6. G815LR 参考样本的使用方法

参考样本用于界定“哪些绝对值不能简单判错”，不是建立统计学正常区间。当前公开证据包括：

- Intel 官方：275HX PBP 55W、MTP 160W、最高工作温度 105°C。
- NVIDIA 官方：RTX 5070 Ti Laptop GPU 60–115W；具体整机可配置 Dynamic Boost。
- ASUS 官方：G815LR 的 RTX 5070 Ti 最高 140W，CPU 使用 Conductonaut Extreme。
- LaptopMedia 精确型号评测：CPU 长负载约 146W/89°C，GPU 游戏约 138W/71°C 平均、74°C 峰值。
- iXBT 同 G815 机身评测：CPU 单烤约 93W/98°C，双烤 CPU 约 58W/93°C，证明双烤低 CPU 瓦数可能是正常分配结果。
- 精确型号异常前后个案：高空闲/轻载温度、约 40W 高温与性能下降，维修后明显改善；仅为 D 级单案例。
